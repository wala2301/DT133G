Dependency management in software projects involves tracking, updating, and securing third-party libraries and tools that a project relies on.
A dependency manifest (requirements.txt, package.json, pyproject.toml) declares the packages needed and their version constraints. Pinning exact versions (==1.2.3) produces reproducible builds but requires explicit updates. Using compatible release constraints (>=1.2,<2) allows automatic minor and patch updates while preventing breaking changes.
Lock files (requirements.txt generated by pip-compile, package-lock.json, poetry.lock) record the exact resolved versions of all direct and transitive dependencies at a point in time. Committing the lock file to version control ensures that all developers and CI environments use identical dependency trees.
Security scanning tools such as pip-audit, safety, and Dependabot check installed packages against known vulnerability databases and alert when a dependency has a published CVE. Regular dependency updates, ideally automated, reduce the window of exposure to known vulnerabilities and prevent large accumulation of outdated packages that are harder to upgrade all at once.
