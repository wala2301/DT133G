Python's functools module provides higher-order functions that work on or return other functions, supporting functional programming patterns.
functools.lru_cache caches the return values of a function keyed on its arguments, so repeated calls with the same arguments return the cached result without recomputation. The maxsize parameter controls how many entries are cached; setting it to None creates an unbounded cache.
functools.partial creates a new callable from an existing function with some arguments pre-filled, which is useful for adapting a function to a specific context without writing a wrapper. functools.reduce applies a binary function cumulatively to an iterable, for example to compute a product or concatenate strings.
functools.wraps is used inside decorators to copy the wrapped function's metadata to the wrapper function, ensuring that __name__, __qualname__, __doc__, and __annotations__ are preserved. functools.total_ordering fills in missing comparison methods on a class that defines __eq__ and one of __lt__, __le__, __gt__, or __ge__.
