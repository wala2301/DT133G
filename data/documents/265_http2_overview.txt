HTTP/2 is a major revision of the HTTP protocol designed to reduce latency and improve throughput by changing how messages are transmitted, while keeping the same semantics as HTTP/1.1.
HTTP/2 uses a binary framing layer instead of the text-based format of HTTP/1.1. This enables multiplexing: multiple requests and responses can be interleaved on a single TCP connection simultaneously, eliminating the head-of-line blocking problem of HTTP/1.1 pipelines.
Header compression using HPACK reduces the overhead of repeated headers across requests in the same connection. Server push allows servers to proactively send resources before the client requests them, though this feature is now less commonly used due to caching complexities.
HTTP/2 requires TLS in practice (though not in the specification) because browsers only implement HTTP/2 over encrypted connections. Server and API frameworks such as nginx, uvicorn, and most cloud load balancers support HTTP/2 transparently, meaning application code typically does not need to change to benefit from it.
