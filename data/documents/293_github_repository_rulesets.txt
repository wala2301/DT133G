GitHub repository rulesets allow organisations and repository administrators to define and enforce rules across multiple repositories from a single configuration.
Rulesets replace and extend branch protection rules with a more flexible targeting model. A ruleset can target branches or tags matched by name patterns and can be applied to an entire organisation, a subset of repositories identified by name or topic, or a single repository.
Rules within a ruleset include requiring pull request reviews, requiring status checks to pass, blocking force pushes, restricting deletions, and requiring signed commits. Each rule can be set to active, evaluate (reports violations without blocking), or disabled, allowing gradual rollout.
Bypass lists allow specific roles, teams, or apps to bypass the ruleset's restrictions, for example to allow a release automation bot to merge without a review. Ruleset insights provide a view of how often rules are triggered and bypassed across all targeted repositories, enabling administrators to audit compliance.
