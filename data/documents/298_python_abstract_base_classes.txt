Python's abc module provides tools for defining abstract base classes (ABCs), which specify interfaces that subclasses are required to implement.
An abstract base class is created by inheriting from ABC and marking methods with the @abstractmethod decorator. Attempting to instantiate a class that inherits from an ABC without implementing all abstract methods raises TypeError, enforcing the interface at class creation time.
ABCs are used to define shared interfaces for interchangeable components. For example, a retrieval system might define an abstract BaseRetriever class with an abstract retrieve(query) method. Each retrieval strategy (TF-IDF, BM25, dense) inherits from BaseRetriever and provides its own implementation, allowing them to be used interchangeably.
The abc module also provides abstract class methods and abstract static methods using @abstractclassmethod and @abstractstaticmethod. Abstract properties are defined by combining @property and @abstractmethod. ABCs complement duck typing in Python: they enforce structural contracts at class definition time rather than at call time.
