GraphQL pagination allows clients to retrieve large collections of data in manageable pages rather than receiving the entire dataset in one response.
GitHub's GraphQL API uses cursor-based pagination, also called connection-based pagination, which is more stable than offset pagination when items are added or removed between pages. A connection type wraps a list of edges; each edge contains a node (the item) and a cursor (an opaque string identifying that item's position).
The first and after arguments fetch the next page: first specifies how many items to return, and after takes the cursor of the last item from the previous page. The pageInfo object in the response contains hasNextPage and endCursor fields. Clients keep requesting the next page until hasNextPage is false.
Backward pagination uses last and before in the same way. Cursor-based pagination provides consistent results even when items are inserted or deleted between requests, unlike offset-based pagination where a deleted item causes subsequent pages to shift, leading to duplicate or skipped results.
