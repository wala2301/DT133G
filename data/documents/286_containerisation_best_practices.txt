Containerisation packages an application and its dependencies into a container image that runs consistently across different environments.
Docker is the most widely used containerisation platform. A Dockerfile defines the image build process: starting from a base image, copying application files, installing dependencies, and specifying the command to run. docker build creates the image; docker run starts a container from it.
Containers are isolated from each other and the host using Linux namespaces and cgroups. Each container has its own filesystem, process space, and network interface, but shares the host OS kernel, making containers more lightweight than virtual machines.
Container images should be kept small to reduce attack surface and image pull time. Best practices include using minimal base images (such as python:3.12-slim or distroless), running as a non-root user, not storing secrets in the image, and using multi-stage builds to exclude build-time tools from the final image. Images should be scanned for vulnerabilities as part of the CI pipeline before deployment.
