NoSQL databases store and retrieve data using models other than the relational tables of SQL databases, trading some query flexibility for scalability and schema flexibility.
Document stores such as MongoDB and CouchDB store data as JSON-like documents grouped in collections. Each document can have a different structure, which suits data with variable or evolving schemas. Documents are retrieved and updated as a whole, which aligns well with object-oriented code.
Key-value stores such as Redis and DynamoDB map keys to values with minimal structure. They are extremely fast for simple lookups and are commonly used for caching, session storage, and rate limiting. Wide-column stores such as Apache Cassandra organise data in rows and columns but allow different rows to have different columns, optimising for high write throughput at scale.
Graph databases such as Neo4j represent data as nodes and edges, making them efficient for queries that traverse relationships. NoSQL databases are not a replacement for SQL; the choice depends on the data model, query patterns, and consistency requirements of the application.
