Code formatters automatically apply a consistent style to source code, eliminating debates about formatting in code review and ensuring a uniform codebase.
Black is the standard Python code formatter. It reformats code according to a single opinionated style with minimal configuration. Running black on a file rewrites it in place. Black is deterministic: the same input always produces the same output, and formatting is idempotent.
isort sorts and groups import statements alphabetically and by type (standard library, third-party, local). It can be configured to be compatible with Black. Together, Black and isort cover the two most common style concerns in Python files.
Formatters should be run automatically as a pre-commit hook using the pre-commit framework, and as a CI check that fails the pipeline if files are not formatted. This ensures formatting is enforced consistently without requiring manual effort from developers. For JavaScript and other languages, Prettier fills a similar role to Black.
