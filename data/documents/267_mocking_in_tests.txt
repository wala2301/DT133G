Mocking replaces real dependencies with controlled substitutes during testing, allowing units of code to be tested in isolation.
Python's unittest.mock module provides Mock and MagicMock objects that record calls and can be configured to return specific values or raise exceptions. The patch() function and decorator temporarily replace an object in a module's namespace for the duration of a test.
Common targets for mocking include external HTTP requests (to avoid network calls in tests), database queries, file I/O, and time functions. For HTTP calls, the responses library for the requests library and httpretty for broader HTTP mocking are popular alternatives to manually patching.
Mocks should assert not only return values but also that the mock was called the expected number of times with the expected arguments, using assert_called_once_with() or assert_any_call(). Over-mocking can lead to tests that pass even when the production code is broken, so integration tests that use real dependencies should complement unit tests that rely on mocks.
