Python's enum module provides a way to define named constants grouped under a class, improving code readability and preventing invalid values.
An enumeration is defined by inheriting from Enum. Each member is a name-value pair accessible as ClassName.MEMBER_NAME. Enum members have a .name attribute (the string name) and a .value attribute (the associated value). Iterating over an Enum class yields all its members in definition order.
IntEnum inherits from both int and Enum, making members directly comparable to integers. StrEnum (Python 3.11+) inherits from str, making members usable wherever strings are expected. Flag and IntFlag support bitwise combination of members, useful for representing sets of options.
Enums are preferred over bare string or integer constants because they prevent typos (misspelled strings are not members of the enum), enable IDE autocompletion, and make the valid values explicit in the code. They are particularly useful for state machines, configuration options, and API response codes where a fixed set of values is expected.
