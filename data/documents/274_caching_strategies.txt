Application-level caching stores the results of expensive computations or slow external calls so that subsequent requests for the same data can be served from memory instead of recomputing them.
In-memory caches such as Redis and Memcached store key-value pairs with optional expiry times (TTL). When a cache miss occurs, the application fetches the data from the source, stores it in the cache, and serves the response. On the next request for the same key, the cache serves the stored value directly.
Cache invalidation is the hardest problem in caching. A cached value becomes stale when the underlying data changes. Strategies include time-based expiry (TTL), explicit invalidation on write, and cache-aside (the application manages reads and writes), write-through (writing to the cache and the database simultaneously), and write-behind (buffering writes in the cache before flushing to the database).
In Python applications, functools.lru_cache and functools.cache provide in-process memoisation for pure functions. These are suitable for caching results of CPU-bound computations within a single process but do not survive process restarts or share state across multiple processes.
