Server-Sent Events (SSE) is a simple protocol for pushing updates from a server to a browser client over a standard HTTP connection.
The client opens a connection to an SSE endpoint using the EventSource API in the browser. The server keeps the connection open and sends text messages in the format data: <payload>\n\n. The browser fires a message event for each received payload.
Unlike WebSockets, SSE is unidirectional: only the server sends messages. This makes it simpler to implement and suitable for use cases such as live notifications, progress updates, and streaming log output where the client does not need to send data after the initial request.
SSE connections automatically reconnect if interrupted, using the Last-Event-ID header to resume from the last received event ID. This built-in reconnection and ordering support distinguishes SSE from plain long-polling. FastAPI and other async frameworks support SSE via streaming responses using the StreamingResponse class.
