Code smells are surface indicators in source code that suggest a deeper design problem, even when the code works correctly.
Long methods are a common smell: a function that spans hundreds of lines is difficult to understand and test. Divergent change occurs when a single class must be modified for multiple unrelated reasons, suggesting the class has too many responsibilities. Shotgun surgery is the inverse: a single change requires modifications to many different classes.
Data clumps are groups of data items that always appear together, such as a street, city, and postcode. These often belong together in their own class. Primitive obsession is the excessive use of primitive types (strings, integers) where a dedicated class or enum would better capture the concept and enforce constraints.
Feature envy describes a method that accesses data from another class more than its own, which usually means the method belongs in the other class. Identifying these smells and addressing them through targeted refactoring leads to code that is easier to change, test, and understand.
