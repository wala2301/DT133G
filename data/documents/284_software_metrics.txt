Software metrics are quantitative measures of software properties used to assess quality, track progress, and identify areas needing improvement.
Cyclomatic complexity measures the number of independent paths through a function, which correlates with testability and defect rate. A function with cyclomatic complexity above 10 is considered complex and a candidate for refactoring. Test coverage measures the percentage of lines, branches, or paths executed by the test suite.
Code churn, the rate at which files change over time, is a predictor of defects: frequently changed files accumulate debt and are more likely to contain bugs. Coupling and cohesion metrics assess how tightly modules depend on each other and how focused their responsibilities are.
Metrics should be used as indicators rather than targets. When a metric becomes a goal, developers optimise the metric rather than the underlying quality. For example, high test coverage does not guarantee meaningful tests. Metrics are most useful when interpreted alongside code review and contextual knowledge of the codebase.
