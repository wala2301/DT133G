GraphQL is a query language for APIs that lets clients specify exactly the data they need, reducing over-fetching and under-fetching compared to REST.
A GraphQL API is defined by a schema that describes the types, fields, and relationships in the data model. The schema is written in the Schema Definition Language (SDL). Queries are validated against the schema before execution, catching errors at the API boundary.
Queries fetch data and look like the shape of the response. Mutations modify data and also return a defined type. Subscriptions establish a persistent connection and push updates to clients when data changes. All three operation types are sent as POST requests to a single endpoint.
GraphQL resolvers are server-side functions that return the data for each field. Because clients select exactly the fields they need, the server only calls the resolvers required to satisfy the query, and responses contain no extraneous fields. This field selection also makes GraphQL APIs self-documenting through introspection queries.
