Context managers in Python manage resource acquisition and release using the with statement, ensuring cleanup happens even if an exception is raised.
The with statement calls the __enter__ method of a context manager when the block begins and __exit__ when it ends, regardless of whether the block completes normally or raises an exception. The as clause binds the value returned by __enter__ to a variable.
Common built-in context managers include open() for files, threading.Lock() for thread synchronisation, and unittest.mock.patch() for test patching. The contextlib.contextmanager decorator allows creating context managers using a generator function with a single yield, eliminating the need to write a full class with __enter__ and __exit__.
Context managers are the recommended way to handle resources such as file handles, database connections, and network sockets because they guarantee the resource is released promptly even when errors occur, preventing resource leaks.
