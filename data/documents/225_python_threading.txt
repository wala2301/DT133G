Python threading allows multiple threads to run concurrently within a single process, sharing the same memory space.
The threading module provides the Thread class. A thread is created by passing a target function and optional args, then calling start(). The main thread can wait for a thread to finish by calling join(). The daemon flag makes a thread exit automatically when the main thread exits.
The Global Interpreter Lock (GIL) in CPython prevents more than one thread from executing Python bytecode at a time. This means CPU-bound work does not benefit from threading in CPython; for CPU-bound parallelism, the multiprocessing module or concurrent.futures.ProcessPoolExecutor is preferred instead.
Threading is well suited for I/O-bound tasks such as making HTTP requests, reading files, or waiting on database queries, where threads spend most of their time waiting rather than computing. Lock, RLock, Semaphore, and Event objects from the threading module coordinate access to shared resources and prevent race conditions.
