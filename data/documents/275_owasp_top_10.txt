The OWASP Top 10 is a regularly updated list of the most critical web application security risks, widely used as a reference for security requirements and code review.
Injection flaws, including SQL injection and command injection, occur when untrusted input is interpreted as a command or query. Prevention requires parameterised queries, input validation, and avoiding shell execution with user-supplied data. Broken access control allows users to act beyond their intended permissions; every request should be authorised server-side and object-level access verified.
Cryptographic failures arise from storing sensitive data unencrypted, using weak algorithms, or transmitting data over unencrypted channels. Security misconfiguration is a broad category covering default credentials, unnecessary features enabled, and missing security headers. Insecure deserialization can lead to remote code execution when untrusted data is deserialised.
Using components with known vulnerabilities exposes applications to attacks targeting unpatched libraries. Developers should monitor dependency advisories, update dependencies regularly, and use software composition analysis tools to track vulnerabilities in third-party packages automatically.
