The twelve-factor app methodology is a set of principles for building software-as-a-service applications that are portable, scalable, and maintainable.
Codebase: one codebase tracked in version control, deployed to multiple environments. Dependencies: explicitly declare and isolate dependencies using a manifest file; never rely on system-wide packages. Config: store configuration in environment variables, not in the codebase, so the same build can run in development, staging, and production.
Backing services such as databases, queues, and caches are treated as attached resources accessed via URLs in configuration, not hardcoded. Build, release, and run stages are strictly separated. Processes are stateless and share-nothing; persistent state lives in a backing service.
Port binding, concurrency via process model, disposability with fast startup and graceful shutdown, dev/prod parity, logs as event streams, and admin processes as one-off tasks round out the twelve factors. Together they describe patterns that enable modern cloud-native deployment and make applications resilient to the failure and replacement of individual instances.
