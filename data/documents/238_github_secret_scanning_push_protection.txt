Secret scanning push protection blocks commits containing recognised secret patterns before they are pushed to a repository.
When push protection is enabled and a developer attempts to push a commit containing a token, API key, or credential matching a supported pattern, GitHub rejects the push and displays a message identifying which secret was detected.
Developers can bypass the block by providing a reason (the secret is a test value, it is already in the environment, or the risk is acceptable), but each bypass is logged and visible to repository administrators.
Push protection covers more than 200 token types from providers including AWS, Azure, Google Cloud, Stripe, and Slack. Custom patterns can be defined using regular expressions to detect organisation-specific secret formats that are not covered by the default ruleset.
