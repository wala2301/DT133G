The GitHub REST API search endpoints allow querying repositories, code, commits, issues, pull requests, topics, and users across GitHub.
Search requests are sent to GET /search/{type} with a q parameter containing the search query. Qualifiers narrow the search: for repositories, qualifiers include language:python, stars:>1000, or user:octocat. For code search, the qualifier repo: scopes results to a specific repository.
Results are paginated and include a total_count field showing the total number of matching items. The items array contains the matched objects with relevance-ranked results. Search is rate-limited more strictly than other endpoints: authenticated requests allow 30 requests per minute for most types.
The code search endpoint requires authentication and does not return results for forks by default. The text-match media type (application/vnd.github.text-match+json) can be requested to receive highlighted fragments showing where the query matched in each result.
