Python's sys module provides access to interpreter-level parameters and functions used to interact with the Python runtime.
sys.argv is a list of command-line arguments passed to the script, where sys.argv[0] is the script name and subsequent elements are the arguments. For structured argument parsing, argparse is recommended over reading sys.argv directly.
sys.path is the list of directories Python searches when importing modules. Appending a directory to sys.path at runtime makes modules in that directory importable, though this should be avoided in favour of proper packaging. sys.modules is a dictionary of all imported modules; deleting an entry forces a fresh import on the next import statement.
sys.exit(code) raises SystemExit with the given code, terminating the interpreter. An exit code of 0 indicates success; non-zero codes indicate failure. sys.stderr and sys.stdout are file-like objects for writing to standard error and standard output respectively, which is useful for diagnostic messages in scripts.
