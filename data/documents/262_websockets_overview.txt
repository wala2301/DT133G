WebSockets provide a full-duplex communication channel over a single, long-lived TCP connection, enabling servers to push data to clients without polling.
The WebSocket handshake begins as an HTTP/1.1 upgrade request. The client sends an Upgrade: websocket header and a Sec-WebSocket-Key; the server responds with 101 Switching Protocols and a Sec-WebSocket-Accept confirming the key. After the handshake, the connection switches to the WebSocket framing protocol.
Once established, either endpoint can send messages at any time without the overhead of HTTP headers on each message. Messages can be text (UTF-8 strings) or binary frames. The connection remains open until either side sends a close frame.
WebSockets are well suited for real-time applications such as chat, live dashboards, collaborative editing, and streaming notifications. For applications where the server only needs to push updates (not receive from the client), Server-Sent Events are a simpler alternative that works over standard HTTP.
