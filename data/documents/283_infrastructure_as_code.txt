Infrastructure as Code (IaC) manages and provisions computing infrastructure through machine-readable configuration files rather than manual processes.
IaC tools such as Terraform, Pulumi, and AWS CloudFormation describe the desired state of infrastructure (virtual machines, networks, databases, load balancers) in configuration files. The tool reconciles the actual state with the desired state, creating, modifying, or deleting resources as needed.
Treating infrastructure as code enables the same practices used for application code: version control, code review, automated testing, and consistent deployments across environments. Infrastructure configurations stored in Git provide an audit trail of changes and enable rollback by reverting commits.
Declarative IaC tools (Terraform, CloudFormation) describe the desired end state and let the tool determine the required actions. Imperative tools (Ansible for configuration management) specify the steps to execute. Both approaches eliminate configuration drift caused by manual changes to servers and ensure that environments can be recreated reliably from the configuration files alone.
