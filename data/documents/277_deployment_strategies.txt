Software deployment strategies control how new versions of an application are released to users, balancing speed, risk, and rollback capability.
A blue/green deployment maintains two identical production environments. The new version is deployed to the inactive environment (green), tested, and then traffic is switched over instantly. If problems arise, traffic can be switched back to the old environment (blue) immediately with no downtime.
A canary release gradually shifts a small percentage of traffic to the new version, monitoring error rates and performance before rolling it out to all users. This limits the blast radius of a bug to only the canary population and allows data-driven decisions about whether to proceed or roll back.
Rolling deployments replace instances of the old version with the new version incrementally, maintaining capacity throughout. Feature flags decouple deployment from release, allowing new code to be deployed to production while the feature is hidden behind a flag and only enabled for specific users or groups.
