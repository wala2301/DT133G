Git stash temporarily shelves changes in the working directory that are not ready to commit, allowing the developer to switch context without losing work.
git stash saves all tracked modifications and staged changes to a stack and restores the working directory to the last commit. git stash pop reapplies the most recently stashed changes and removes them from the stack. git stash apply reapplies without removing, useful when the same stash needs to be applied to multiple branches.
Multiple stashes can be managed with git stash list, which shows all stashed entries with their index. A specific stash is applied with git stash apply stash@{n}. Stashes can be given descriptive names using git stash push -m 'description'.
Stash is particularly useful when an urgent bug is reported while working on a feature: the in-progress work is stashed, the hotfix branch is checked out and fixed, and then the stash is popped to resume the feature. Stashes are local and are not pushed to the remote repository.
