Event-driven architecture (EDA) is a design pattern where components communicate by producing and consuming events rather than calling each other directly.
An event represents something that happened: an order was placed, a file was uploaded, or a sensor reading was recorded. Producers publish events to an event broker or message bus without knowing which consumers will process them. Consumers subscribe to event types and react when matching events arrive.
This decoupling improves scalability because producers and consumers can be scaled independently, and resilience because a slow or failed consumer does not block the producer. New consumers can be added without modifying existing producers, making the system extensible.
Common brokers used in event-driven systems include Apache Kafka, RabbitMQ, AWS EventBridge, and Redis Streams. Event-driven patterns are well suited to microservice architectures where services need to react to state changes in other services without tight synchronous coupling.
