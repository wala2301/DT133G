Python's built-in logging module provides a flexible framework for emitting log messages
from applications and libraries. The module defines a hierarchy of loggers identified by
dotted names, and log records propagate up the hierarchy unless propagation is disabled.
Handlers determine where log records are sent, such as to the console via StreamHandler
or to a file via FileHandler, and formatters control the string representation of each
record. The recommended pattern for library code is to create a module-level logger with
logging.getLogger(__name__) and avoid configuring handlers, leaving that to the
application developer.
