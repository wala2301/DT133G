Static code analysis examines source code without executing it, detecting bugs, security vulnerabilities, style violations, and code quality issues automatically.
Linters enforce coding style and flag common mistakes. For Python, flake8 and pylint check for style violations, undefined variables, unused imports, and complexity. Ruff is a modern, fast alternative that combines many linting rules in a single tool.
Security-focused static analysis tools such as Bandit for Python scan for patterns associated with known vulnerabilities, including hardcoded passwords, use of insecure functions like eval(), and weak cryptographic configurations.
Type checkers such as mypy and pyright perform static analysis of type annotations to catch type errors before runtime. They are particularly effective when combined with comprehensive type annotations and are integrated into CI pipelines to prevent merging code that introduces type errors. Static analysis is most effective when integrated into the editor and the CI pipeline so that feedback is immediate and issues do not accumulate.
