GitHub webhook payloads are JSON objects delivered via HTTP POST to a configured endpoint whenever a subscribed event occurs in a repository or organisation.
Each payload includes a common set of fields: the action field describes what happened (e.g., opened, closed, labeled), the sender object identifies the user who triggered the event, and the repository object describes the repository where the event occurred. Event-specific fields then carry the relevant resource data.
Payloads are delivered with a Content-Type of application/json or application/x-www-form-urlencoded depending on the configuration. The X-GitHub-Event header identifies the event type (push, pull_request, issues, etc.) and X-GitHub-Delivery contains a unique UUID for the delivery.
Large payloads, such as those for push events with many commits, may be truncated. Webhooks have a 10-second delivery timeout; if the endpoint does not respond with a 2xx status within that window, GitHub schedules a retry with exponential backoff.
